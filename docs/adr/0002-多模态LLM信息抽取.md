# 多模态 LLM 而非纯文本 LLM

信息抽取阶段将原图与 OCR 文字一同发送给多模态大模型（而非仅发送 OCR 文本给纯文本模型）。

**为什么 OCR 文字不够？** 增值税发票表格结构复杂，OCR 在印章重叠、表格线干扰、字段错行时可能输出乱序文本。纯文本模型只能做模式匹配，无法利用原始版面的视觉线索（列对齐、框线位置、印章与文字的遮挡关系）来纠错。多模态模型同时看图和文字，在这些边缘场景下准确率显著更高。

**代价**：多模态 API 调用延迟更长、费用更高。

**备选方案**：纯文本 LLM（仅发送 OCR 输出文本）。否决原因：当 OCR 输出质量差时，纯文本模型没有可用的纠错信号，误提取率无法接受。但该方案作为 LLM 不可用时的降级策略保留（换为正则规则提取）。
