# 双信息校验架构

验票流程不信任单一数据源：二维码解析和 OCR+LLM 两路独立提取发票字段，逐字段比对后才允许进入报销表单。任一可校验字段不一致则阻塞提交。

**为什么不是直接信任 OCR？** 增值税发票二维码是官方签发的结构化数据，准确率接近 100%，而 OCR 在图像模糊、倾斜、印章遮挡下存在误读风险。两路交叉校验用极低的工程代价换一道 OCR 误读的安全网。

**代价**：多了 YOLO 检测 + 二维码解析的流水线节点，且必须处理二维码缺检的降级路径。

**备选方案**：纯 OCR+LLM 单路提取（省掉二维码环节）。否决原因：失去了唯一的真实数据校对源，发票号码误读会直接导致重复报销检测失效。
