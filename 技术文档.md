# 技术需求文档：智能发票识别与自动报销填单系统

本技术文档旨在明确“深度学习理论及应用实践”课程大作业的项目目标、核心功能逻辑及技术实现规范。该系统通过集成计算机视觉与大语言模型技术，实现从发票上传到自动化入库的全流程管理。

---

## 1. 项目目标
本项目的核心目标是开发一个工程化、可演示的智能报销小系统。系统旨在利用多种技术栈解决电子发票报销中人工录入效率低、重复报销等痛点：
* **核心流程**：实现发票上传、二维码检测解析、文字识别、信息提取、双信息校验、自动填单、重复检测及存储的全流程自动化。
* **技术栈**：涵盖目标检测、OCR、大模型信息抽取、二维码解析、Web 开发及数据库技术。

---

## 2. 核心功能需求

### 2.1 Web 交互界面
* **功能支持**：支持发票图片上传、识别结果展示、表单预览。
* **提示机制**：系统通过弹窗提示校验结果（通过/异常）或报警重复报销。

### 2.2 二维码检测与校验
* **定位裁剪**：使用 YOLO 模型定位并裁剪二维码区域。
* **解析规范**：严禁直接全图解析，必须针对裁剪出的二维码区域进行解析。
* **双信息校验**：将二维码数据与 OCR 及大模型提取的信息进行比对校验，输出一致性提示。

### 2.3 文本提取与自动化填单
* **PaddleOCR 离线识别**：下载推理模型实现本地离线 OCR，禁止使用在线识别接口。
* **LLM 信息抽取**：调用在线大模型 API 提取发票号码、日期、购买方、销售方、金额、税额、价税合计等关键字段。
* **自动填单**：校验通过后，信息自动填入标准报销表单。

### 2.4 数据库重复报销检测
* **存储查重**：存储已报销发票号，上传新发票时自动检索。
* **防御机制**：检测到重复则报警并阻止提交，不重复则准许存入数据库。

## 3. 技术规范要求

| 模块名称 | 技术规范要求 |
| :--- | :--- |
| **YOLO** | 必须执行“检测 + 裁剪”流程。 |
| **PaddleOCR** | 必须进行本地离线推理。 |
| **二维码** | 必须解析并与 OCR 结果进行相互校验。 |
| **大模型** | 仅限使用在线 API 执行信息抽取任务。 |
| **数据库** | 选用 SQLite 或 MySQL（推荐轻量级部署）。 |
| **代码工程** | 必须模块化、注释清晰，并提供 requirements.txt。 |
